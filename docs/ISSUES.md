# 작업 이슈 — 수직 슬라이스 (finalstudy)

> 원칙: 각 슬라이스는 **DB(저장)·API·UI를 관통하는 end-to-end 기능**이다. 가로 레이어(예: "백엔드 먼저, UI 나중")로 나누지 않는다. 위에서부터 순서대로 구현하면 매 단계마다 동작하는 제품이 남는다.

---

## Slice 0 — 프로젝트 부트스트랩 & 빈 노트 화면
**무엇을 만드나**
- Next.js(App Router) + TS + Tailwind 초기화.
- `.env.example`에 `GEMINI_API_KEY` 정의, 서버 전용 로딩.
- 루트 페이지: 업로드 영역(아직 비활성/목업) + 빈 노트 뷰 자리.
- 공통 레이아웃/타이포(노션풍) 베이스.

**완료 기준 (AC)**
- `npm run dev`로 앱이 뜨고 루트 화면이 보인다.
- 빌드/린트 통과.
- 환경변수 누락 시 명확한 안내(런타임 에러 가드).

**의존성**: 없음 (시작점)

---

## Slice 1 — 텍스트 대본 → 구조화 노트 → 화면 렌더 (핵심 수직선)
> STT를 잠시 건너뛰고, **대본 텍스트 입력**으로 전체 파이프라인을 먼저 관통시켜 핵심 가치를 가장 빨리 검증한다.

**무엇을 만드나**
- UI: 대본 붙여넣기 입력 + "노트 생성" 버튼.
- API: `POST /api/generate` — 대본 텍스트를 받아 Gemini로 **Note JSON**(PRD 8장 스키마) 생성. 서버에서만 `GEMINI_API_KEY` 사용.
- UI: 반환된 Note JSON을 렌더 (heading/paragraph 우선).
- 저장: 생성 결과를 localStorage에 저장(임시 단일 노트).

**완료 기준 (AC)**
- 대본을 넣고 버튼을 누르면 Gemini가 스키마에 맞는 JSON을 반환한다.
- 스키마 검증 실패 시 사용자에게 오류 메시지를 보여준다.
- 생성된 노트의 제목/계층 섹션이 화면에 표시된다.
- 새로고침해도 마지막 노트가 복원된다.

**의존성**: Slice 0

---

## Slice 2 — 오디오 업로드 → STT → 노트 생성 (입력 소스 완성)
**무엇을 만드나**
- UI: 오디오 파일 선택(mp3/m4a/wav), ≤50MB 클라이언트 검증, 진행 상태 표시.
- API: `POST /api/note-from-audio` — 파일 → Gemini 오디오 입력 → Slice 1 구조화 파이프라인 연결.
- 에러: 용량 초과/형식 오류/STT 실패 처리.

**완료 기준 (AC)**
- 20~60분(≤50MB) 오디오 업로드 시 노트가 생성된다.
- 50MB 초과 파일은 업로드 전 거절되고 안내가 뜬다.
- 처리 중 진행 상태(전사 중/구조화 중)가 보인다.

**의존성**: Slice 1

---

## Slice 3 — 능동 암기 토글 블록
**무엇을 만드나**
- AI 프롬프트에 `toggle` 블록 생성 지침 추가(개념 가림/정답).
- UI: 토글 블록 컴포넌트 — 기본 접힘, 클릭 시 정답 펼침.
- localStorage 스키마에 toggle 포함 반영.

**완료 기준 (AC)**
- 노트에 토글 블록이 최소 1개 포함되며 기본 접힘 상태다.
- 클릭 시 정답이 펼쳐지고 다시 접을 수 있다.
- PDF 출력 전이라도 화면에서 정상 동작한다.

**의존성**: Slice 1 (오디오와 무관하게 병행 가능)

---

## Slice 4 — Mermaid 순서도 블록
**무엇을 만드나**
- AI 프롬프트에 `mermaid` 블록(메커니즘/프로세스) 생성 지침 추가.
- UI: Mermaid 렌더 컴포넌트(클라이언트 사이드), 렌더 실패 시 코드 폴백 표시.

**완료 기준 (AC)**
- 메커니즘성 내용이 있으면 노트에 순서도가 그려진다.
- 잘못된 mermaid 문법일 때 앱이 깨지지 않고 폴백을 보여준다.

**의존성**: Slice 1

---

## Slice 5 — 비교 표 블록
**무엇을 만드나**
- AI 프롬프트에 `table` 블록(비교용) 생성 지침 추가.
- UI: 표 렌더 컴포넌트(노션풍 스타일).

**완료 기준 (AC)**
- 비교 가능한 정보가 있으면 표로 렌더된다.
- 헤더/행 구조가 깨지지 않는다.

**의존성**: Slice 1

---

## Slice 6 — 출제 힌트 자동 동기화 (배지)
**무엇을 만드나**
- AI 프롬프트에 출제 단서("중요", "시험에 낸다" 등) 탐지 → 해당 블록 `examHint: true` 지침 추가.
- UI: `examHint`가 true인 블록에 `강조·출제유력` 배지 표시 + 강조 스타일.
- (선택) 출제유력 항목만 모아 보는 필터/요약 섹션.

**완료 기준 (AC)**
- 출제 단서가 담긴 대본에서 해당 개념 블록에 배지가 붙는다.
- 배지가 시각적으로 명확히 구분된다.

**의존성**: Slice 1 (모든 블록 타입에 적용되므로 Slice 3~5 이후 권장)

---

## Slice 7 — 노트 라이브러리 (다중 노트 저장/관리)
**무엇을 만드나**
- localStorage를 단일 노트 → **노트 목록**으로 확장.
- UI: 사이드바/목록에서 지난 노트 열기·삭제·제목 표시.

**완료 기준 (AC)**
- 여러 노트를 생성·저장하고 목록에서 선택해 다시 열 수 있다.
- 삭제가 동작하고 새로고침 후에도 목록이 유지된다.

**의존성**: Slice 1

---

## Slice 8 — PDF 내보내기
**무엇을 만드나**
- Browser print 기반 PDF 저장.
- 출력 전용 CSS: 사이드바/입력/버튼 숨김, 노트만 표시.
- 닫힌 토글도 PDF에서는 펼친 상태로 출력.

**완료 기준 (AC)**
- PDF 미리보기에서 노트 내용만 보인다.
- 표/순서도/토글 내용이 정상 출력된다.
- 모든 토글이 펼쳐져 있다.

**의존성**: Slice 3~6 (변환 대상 블록 타입 완성 후)

---

## 권장 구현 순서
`0 → 1 → 2`로 핵심 수직선(오디오→노트)을 먼저 완성한 뒤, `3 → 4 → 5 → 6`으로 노트 포맷(토글/순서도/표/출제힌트)을 채우고, `7 → 8`로 보존·내보내기를 마감한다. Slice 3~5는 서로 독립적이라 병행 가능.
