# PRD — AI 강의 노트 생성기 (finalstudy)

## 1. 개요
강의 녹음 파일을 업로드하면, 학생이 노션에 직접 정리한 듯한 **완벽히 구조화된 능동 암기형 노트**를 한 번에 생성하는 웹 앱. 교수가 음성으로 흘린 출제 힌트까지 자동으로 잡아 노트에 표시한다.

## 2. 해결할 문제 (Problem Statement)
시험 기간 학생들은 강의 녹음을 다시 듣고, 받아쓰고, 핵심을 추리고, 암기용으로 재가공하는 데 막대한 시간을 쓴다. 기존 AI 요약 툴은 **줄글 요약**만 던져줘서, 학생이 다시 구조화·암기 포맷으로 2차 가공해야 한다. 또한 교수가 수업 중 "이거 시험에 낸다"라고 말한 **출제 힌트는 녹음 어딘가에 묻혀** 다시 찾기 어렵다.

핵심 페인 포인트:
1. **2차 가공 피로** — 요약을 받아도 암기 가능한 구조로 다시 만들어야 함.
2. **출제 힌트 유실** — 교수의 구두 강조 단서가 흩어져 사라짐.
3. **비효율적 암기 포맷** — 줄글은 능동 인출(active recall)에 부적합.

## 3. 목표 (Goals)
- 오디오 업로드 → 구조화된 노트까지 **한 번의 흐름**으로 완성.
- 줄글이 아닌 **계층 구조 + 토글 + 표 + 순서도**로 즉시 사용 가능한 노트 제공.
- 교수 출제 힌트를 자동 탐지해 해당 개념에 **`강조·출제유력` 배지** 부착.
- 능동 암기에 최적화된 **토글 인출 테스트** 제공.

## 4. 비목표 (Non-Goals)
- 로그인/계정, 서버 DB, 기기 간 동기화·공유.
- 실시간 마이크 녹음.
- 4MB 초과(긴 강의)의 오디오 분할(chunking) 처리.
- 사용자별 API 키 입력 UI (서버 단일 키 사용).
- 노션 API 직접 연동(자동 push).
- 모바일 네이티브 앱.

## 5. 사용자 스토리 (User Stories)

### US-1 · 노트 자동 생성
> **시험을 앞둔 학생**으로서, **강의 녹음 파일을 업로드**하고 싶다. 왜냐하면 **직접 받아쓰고 정리하는 시간을 없애고 바로 공부하고 싶기** 때문이다.
- 오디오(mp3/m4a/wav, ≤4MB) 업로드 가능.
- 업로드 후 처리 진행 상태가 보인다.
- 처리 완료 시 구조화 노트가 화면에 렌더링된다.

### US-2 · 완벽한 구조화 (2차 가공 제로)
> **학생**으로서, **줄글이 아닌 계층 구조/표/순서도로 정리된 노트**를 받고 싶다. 왜냐하면 **추가 가공 없이 그대로 암기에 쓰고 싶기** 때문이다.
- 노트는 계층형 섹션/소제목으로 구성된다.
- 메커니즘/프로세스는 Mermaid 순서도로 표현된다.
- 비교 가능한 정보는 표로 표현된다.

### US-3 · 출제 힌트 자동 동기화
> **학생**으로서, **교수가 강조한 내용에 출제유력 표시**가 붙길 원한다. 왜냐하면 **무엇을 집중 공부할지 바로 알고 싶기** 때문이다.
- 대본에서 강조·출제 단서("중요", "시험에 낸다" 등)를 탐지한다.
- 해당 개념 옆에 `강조·출제유력` 배지가 표시된다.

### US-4 · 능동 암기 (Active Recall)
> **학생**으로서, **개념을 가리고 스스로 답을 떠올리는 토글**을 쓰고 싶다. 왜냐하면 **읽기보다 인출 연습이 암기에 효과적이기** 때문이다.
- 토글 블록은 기본 접힘(개념/정답 가림) 상태다.
- 클릭하면 내용이 펼쳐져 정답을 확인할 수 있다.

### US-5 · 노트 보존 및 재방문
> **학생**으로서, **생성한 노트를 다시 열어보고** 싶다. 왜냐하면 **비싼 재처리 없이 이어서 공부하고 싶기** 때문이다.
- 생성된 노트는 브라우저 로컬에 저장된다.
- 지난 노트 목록에서 다시 열 수 있다.

### US-6 · 내보내기
> **학생**으로서, **노트를 PDF로 내보내고** 싶다. 왜냐하면 **태블릿/프린트로 바로 공부하고 싶기** 때문이다.
- 노트를 PDF로 저장할 수 있다.
- PDF 출력 시 닫힌 토글도 펼쳐진 상태로 포함된다.

## 6. 기술 스택
- **프레임워크**: Next.js (App Router) + TypeScript
- **스타일**: Tailwind CSS
- **AI**: Gemini API (`@google/genai`)
- **다이어그램**: Mermaid.js
- **저장**: 브라우저 localStorage
- **내보내기**: Browser print 기반 PDF 저장
- **키 관리**: 서버 `.env`의 `GEMINI_API_KEY`, API Route에서만 호출

## 7. 처리 흐름
1. 오디오 업로드(≤4MB)
2. API Route → Gemini로 오디오/텍스트 처리
3. Gemini → 구조화 노트 JSON 생성 (섹션/토글/표/mermaid/출제힌트)
4. 프런트에서 인터랙티브 노트 렌더링
5. localStorage 저장 + PDF 내보내기

## 8. 데이터 모델 (Note JSON 스키마 초안)
```ts
type Note = {
  id: string;
  title: string;
  createdAt: string;
  sourceFileName: string;
  blocks: Block[];
};

type Block =
  | { type: "heading"; level: 1 | 2 | 3; text: string; examHint?: boolean }
  | { type: "paragraph"; text: string; examHint?: boolean }
  | { type: "toggle"; prompt: string; answer: string; examHint?: boolean }
  | { type: "table"; headers: string[]; rows: string[][]; examHint?: boolean }
  | { type: "mermaid"; code: string; caption?: string; examHint?: boolean };
```

## 9. 성공 기준 (MVP)
- 20~30분 강의 오디오 업로드 시 구조화 노트가 생성된다.
- 노트에 토글/표/순서도/출제힌트 배지가 최소 1개 이상 적절히 포함된다.
- 노트가 로컬에 저장되고 재방문 시 복원된다.
- PDF 저장 시 노트 내용과 펼쳐진 토글이 정상 출력된다.
